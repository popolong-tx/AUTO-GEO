# Oracle 客户反馈：舆情分析看板新增需求（2026-06-30）

## 1. 需求背景

客户反馈希望现有 BYDGEO 舆情分析系统从“报告生成型工具”进一步增强为“可视化舆情监测与可追溯分析平台”。核心诉求包括：

- 数据可视化呈现
- 核心传播指标展示
- 文本分析能力增强
- 所有指标与分析结果支持数据源追溯
- 每天两次更新

参考截图中示例主题为 `BYD X Goodwood`，但需求应作为平台通用能力，适用于所有 topic。

---

## 2. 新增功能需求总览

### 2.1 数据可视化呈现

系统需要在每个 topic 的分析结果页增加可视化模块，至少包括：

1. Campaign Overview / 传播概览
   - Total Mentions：总提及量
   - Total Reach：总触达量
   - Total AVE：广告价值等效值
   - Total Social Engagement：社交互动量，如点赞、评论、转发、分享等
   - 与上一周期对比：上涨/下降百分比
   - 小型趋势 sparkline

2. 趋势图
   - Total Mentions Trend：总提及量趋势
   - Mentions and Reach Trend：提及量与触达量组合趋势
   - Sentiment Trend：情绪趋势

3. 情绪分析图
   - Sentiment Breakdown：正面/中性/负面占比环形图
   - Sentiment Trend：按日期展示情绪堆叠柱状图
   - 图表值必须与报告正文和后端 sentiment JSON 一致

4. 地域覆盖图
   - Coverage by Market / Coverage by Country
   - 国家/地区热力地图
   - Countries with Most Coverage 排行榜

5. 来源排行
   - Top Sources by Mentions
   - 按 mentions 排序的媒体/来源列表
   - 支持进入详细来源数据

6. 社交传播排行
   - Coverage Most Shared in Social Media
   - 显示被社媒分享最多的报道
   - 需要展示 X/其他社媒互动信息

7. 潜在浏览量排行
   - Coverage with Highest Potential Views
   - 按潜在 views/reach 排序

---

### 2.2 核心指标与重点关键词

客户明确要求重点展示以下指标：

- Total Mentions
- Total Reach
- Total AVE

建议扩展为统一 KPI 结构：

| 指标 | 中文名 | 说明 |
|---|---|---|
| Total Mentions | 总提及量 | 统计周期内相关报道、社媒贴文、论坛讨论等总提及次数 |
| Total Reach | 总触达量 | 内容可能覆盖的受众规模，优先使用来源平台公开 reach/views 数据 |
| Total AVE | 广告价值等效值 | Advertising Value Equivalency，需明确计算口径 |
| Total Social Engagement | 社交互动量 | 点赞、评论、转发、分享、引用等汇总 |
| Sentiment | 情绪分布 | Positive / Neutral / Negative |
| Top Keywords | 高频关键词 | 与品牌、车型、竞品、事件相关的关键词 |

---

### 2.3 文本分析需求

分析报告和页面需要保留/新增以下文本分析模块：

1. 核心传播主题
   - 本周期主要传播主线
   - 官方叙事、媒体叙事、用户叙事的差异

2. 高频关键词
   - Top keywords
   - 支持按品牌、车型、竞品、风险议题分类

3. 正面及负面报道重点
   - Positive coverage highlights
   - Negative coverage highlights
   - 中性报道也应列明主要关注点

4. 主要媒体观点与潜在风险议题
   - 主流媒体观点摘要
   - 潜在风险、负面叙事、传播扩散风险

5. 品牌、车型及竞品相关讨论
   - BYD 品牌讨论
   - 具体车型讨论
   - 竞品比较，例如 Tesla、Porsche、Rimac、Omoda、Jaecoo 等，按 topic 动态适配

6. 重点新闻摘要
   - 重要新闻列表
   - 每条包含来源、发布时间、摘要、链接、情绪、影响指标

---

### 2.4 数据源可追溯需求

客户重点要求：所有指标和分析结果都应能够点击进入详细数据来源。

每一条来源数据至少包含：

- 原始文章链接
- 媒体名称
- 发布时间
- 国家及语言
- 作者
- Reach
- AVE
- Sentiment
- 文章摘要
- 相关品牌、车型及关键词

建议后端定义统一 SourceEvidence 结构：

### 2.5 更新频率

客户要求：

- 频率：一天两次

建议实现为：

1. 每天自动刷新两次
   - 建议时间：上午 09:00、下午 18:00（可配置）
   - 每次刷新拉取最新数据并生成快照

2. 与现有日快照逻辑协调
   - 普通分析默认读取最近一次当日快照
   - 重新分析按钮仍可强制刷新
   - 若检测到重大事件，应突破当天一致性约束重新分析

3. 页面展示
   - 显示“最近更新时间”
   - 显示“下次计划更新时间”
   - 显示“当前数据周期：Last 7 days / 自定义周期”

## 3. 对现有系统的改造建议

### 3.1 后端改造

新增或增强以下能力：

1. 指标聚合层
   - 从模型输出、搜索结果、来源数据中抽取 KPI
   - 形成标准 metrics JSON

2. 来源证据层
   - 保存每条来源 evidence
   - 提供可点击追溯 API

3. 可视化数据 API
   - `/api/dashboard/{topic_id}`
   - 返回 KPI、趋势、情绪、地域、来源排行、社交传播排行等结构化数据

4. 自动刷新任务
   - 每天两次生成 topic 快照
   - 可复用现有 daily snapshot，但需要支持一天两次版本

5. 报告与可视化一致性
   - 情绪分布图必须使用同一份 sentiment JSON
   - 报告正文中的情绪比例应由后端标准 sentiment 生成或校验

---

### 3.2 前端改造

在 TopicPage 中建议增加：

1. 顶部扁平 KPI 卡片区
   - Total Mentions
   - Total Reach
   - Total AVE
   - Total Social Engagement

2. 可视化 Tabs 或 Grid
   - 趋势图
   - 情绪图
   - 地图/国家排行
   - 来源排行
   - 社交传播排行

3. 数据源抽屉/弹窗
   - 点击 KPI、图表、来源条目可打开 source detail
   - 展示原始链接、媒体、发布时间、国家、语言、作者、Reach、AVE、Sentiment、摘要、关键词

4. 更新状态区
   - 最近更新时间
   - 下次更新时间
   - 一天两次刷新说明

---

## 4. 优先级建议

### P0：必须先做

- KPI 卡片：Total Mentions / Total Reach / Total AVE
- 情绪图与报告值一致
- 社交媒体最新信息恢复稳定展示
- 数据源详情结构定义与 API
- 每条分析结果支持来源追溯

### P1：建议第二阶段做

- 趋势图
- 国家/地区排行
- Top Sources by Mentions
- Coverage Most Shared in Social Media
- 每天两次自动刷新

### P2：增强项

- 地图热力图
- AVE 计算配置
- 竞品/车型/品牌关系图谱
- Dashboard 导出 PDF/Excel

---

## 5. 验收标准

1. 每个 topic 页面能看到核心 KPI：Mentions / Reach / AVE。
2. 情绪分布图和报告正文数值一致。
3. 每个 KPI、图表、分析结论至少能追溯到来源清单。
4. 来源详情包含：原始链接、媒体名称、发布时间、国家语言、作者、Reach、AVE、Sentiment、摘要、品牌/车型/关键词。
5. 社交媒体最新信息不为空；若确实没有数据，必须说明“未检索到”，不能空白。
6. 每天两次刷新任务可配置、可查看最近更新时间。
7. 重新分析按钮能强制刷新并更新当日缓存。

---

## 6. 结论

客户的新需求本质上是把系统从“单次舆情分析报告”升级为“可视化、可追溯、可定时更新的舆情监测平台”。

建议优先落地：

1. 指标结构化；
2. 来源证据结构化；
3. Topic 页面可视化 KPI 和来源追溯；
4. 每天两次刷新；
5. 保证报告、图表、来源三者一致。
